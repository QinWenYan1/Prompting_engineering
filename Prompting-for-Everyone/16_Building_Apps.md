# 📘 16 构建应用 (Building Apps)

> 来源：Andrew Ng | Module 3: Working with Multimedia & Code | 课时 4/6 | ~5 分钟

---

## 🧠 本课核心

你不需要是程序员就能构建软件应用——通过文本 prompt，AI 可以为你生成可玩的游戏、工具 App、交互式网页。关键技巧只有三个字：**目标、输入、输出**。

---

## ✅ 知识点1: 一句话就能生成可玩的应用

**案例 1：障碍物模拟游戏**
```
构建一个游戏，让用户放置障碍物和目标，然后创建你所设计内容的模拟效果。
```
→ Claude 据此创建了一个**可玩的游戏**。
> "只需要这样一个简短的 prompt，顶尖 AI 模型就能创建一个相当有趣的游戏。"

**案例 2：烟花模拟器**
```
生成一个有趣的烟花模拟器。
- 目标：点击屏幕放烟花
- 输入：点击位置
- 输出：彩色烟花动画
```
→ 效果是「一个实际上挺好玩的烟花展示应用」。

**注意点**
- 💡 用 AI 构建应用的门槛正在以月为单位降低——几个月前做不到的事现在能做到了
- ⚠️ Andrew Ng 也提醒：「我不想让这听起来很简单。」——构建好应用仍然需要技巧

---

## ✅ 知识点2: 构建应用的 Prompt 三要素

Andrew Ng 提炼了一个简单框架：

| 要素 | 说明 | 烟花模拟器示例 |
|------|------|-------------|
| **目标（Goal）** | 你想构建什么？ | 「一个有趣的烟花模拟器」 |
| **输入（Inputs）** | 用户如何与应用交互？ | 「点击屏幕」 |
| **输出（Outputs）** | 应用向用户展示什么？ | 「看到彩色烟花展示」 |

**更多实用 App 示例**

| 应用 | Prompt 概述 |
|------|-----------|
| **番茄钟** | 「25 分钟工作 + 5 分钟休息交替的计时器」 |
| **账单计算器** | 「输入总金额和人数，计算每人应付多少」 |
| **穿搭建议器** | 「根据天气推荐穿什么衣服」 |

这些 App 的共同特征：**功能明确、无需外部数据文件、用完即走**。

**注意点**
- 💡 最好的第一个 AI App 是「你打开、用完、关闭，不需要存任何东西」的工具——简单，但有用

---

## ✅ 知识点3: 什么容易构建 vs 什么难——培养直觉

**容易构建**
- 简单的平台跳跃游戏
- 法语单词练习测验
- 单页工具类 App（计算器、计时器、转换器）

**难构建（可能翻车）**
- 多人联网游戏
- 需要 AI 实时反馈的法语口语练习——这需要「live French practice with AI feedback」，不仅仅是单词测验

**Andrew Ng 的建议**
> "最坏的情况就是它没成功——然后你就开始培养什么难、什么容易的直觉了。"

这意味着：**失败本身就是学习**。多试几次，你会越来越清楚什么样的任务 AI 能搞定、什么样的不能。

**注意点**
- 💡 构建 App 的 prompt 工程和文本 prompt 工程遵循同样的原则：给足上下文 + 清晰定义输入输出 + 迭代改进
- 🔄 这和第 9 课的「给 AI 难任务试试看」建议一致——你不知道边界在哪，直到触碰到它

---

## 🔑 本课核心要点

1. 非程序员也能用 AI 构建可用的软件应用——这是真正的能力民主化
2. 构建 App 的 prompt 三要素：目标 × 输入 × 输出
3. 从简单的小工具开始（番茄钟、计算器），逐步培养对 AI 能力的直觉
4. 最坏情况就是失败了——但失败教会你下一次怎么问

## 📌 实践速查

**构建你的第一个 AI App 模板：**
```
我想构建一个 [描述你的 App]。
- 目标：[它要做什么]
- 用户输入：[用户如何操作]
- 期望输出：[用户看到什么/得到什么结果]
请生成一个可以在浏览器中直接运行的完整实现。
```

**适合练手的第一个项目：**
1. 账单分摊计算器
2. 倒计时番茄钟
3. 每日随机决策器（「今天中午吃什么？」）
4. 自定义 Flashcards 记忆卡片
